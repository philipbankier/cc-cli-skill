# Contributing to cc-cli

Thanks for your interest in contributing!

## What We're Looking For

- **New examples** — Multi-agent patterns, CI/CD integrations, interesting use cases
- **Flag documentation updates** — As Claude Code CLI evolves, flags change
- **Bug fixes** — Corrections to code snippets, schemas, or flag descriptions
- **Language coverage** — Code snippets in additional languages (Ruby, Rust, etc.)

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b my-contribution`)
3. Make your changes
4. Test any code examples to ensure they work
5. Submit a pull request with a clear description

## Guidelines

- Keep the skill structure intact — `SKILL.md` is the entry point, guides in `guides/`, reference material in `reference/`
- Code examples should be copy-paste ready and tested
- If adding a new example, include a `WALKTHROUGH.md` explaining the techniques used
- Follow the existing writing style: concise, technical, agent-friendly

## Reporting Issues

Open a GitHub issue for:

- Broken code examples
- Incorrect flag documentation
- Missing coverage of new Claude CLI features
- Suggestions for new examples or guides
